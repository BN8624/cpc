# CPC 작업 컨텍스트

- 이 프로젝트는 설치 없이 사용하는 단일 HTML 도구를 유지한다.
- 원본 `건별_v5.0.html`은 현재 정상 실행되는 기준 버전이다.
- 버전 관리 목적으로 GitHub `BN8624/cpc` 공개 저장소를 생성한다.
- 이후 로직 수정은 별도 작업에서 진행하며, 변경 전후 검증 결과를 기록한다.
- 공개 저장소 주소는 `https://github.com/BN8624/cpc`이다.
- 기본 브랜치는 `main`이며 로컬 `main`이 `origin/main`을 추적한다.
- 이 저장소는 별도 작업 브랜치를 사용하지 않고 항상 `main`에서 작업한다.
- Git 이력으로 이전 버전을 보존하므로 `건별_v5.0.html`을 직접 수정한다.
- 원본 엑셀은 항상 단일 연도이므로 연도 선택·검증 로직은 추가하지 않는다.
- 단일 HTML과 CDN 기반 실행 방식은 유지한다.
- 최소금액·최대금액·단위가 충돌하면 사용자 입력을 자동 보정하지 않고 작업을 중단한다.
- 요청 기간 중 원본 날짜가 없는 기간이 하나라도 있으면 부분 결과를 저장하지 않는다.
- 배분은 고정 반복 횟수 대신 각 날짜의 수용 가능 금액을 계산하는 방식으로 변경한다.
- 잘못된 날짜는 다른 날짜로 보정하지 않고 제외하며 제외 행 수를 로그에 남긴다.
- 날짜·금액 열을 찾지 못하면 임의의 첫 번째 열을 사용하지 않고 명시적으로 중단한다.
- 250원 면세 모드에서는 비활성화된 최대금액 입력의 이전 값을 사용하지 않는다.
- 검증 결과: 배분 경계값 440개 조합 통과, 최소·최대 충돌 오류 확인, 중단 확인, 날짜 엄격 검증 통과.
- 검증 결과: 브라우저 로드 오류 0건, 월별·기수별·과세·면세 UI 전환 확인.
- 검증 결과: SheetJS 0.18.5로 입력 열 인식과 XLSX 생성·재읽기 합계 확인.
